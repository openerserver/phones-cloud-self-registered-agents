# 参与贡献

感谢你帮助改进 Phones Cloud 自注册被控端的公开文档和使用体验。

## 仓库范围

本仓库是客户端分发与公开文档仓库，不包含 Windows、macOS 或 Android 客户端源代码。目前欢迎以下贡献：

- 修正文档中的错误、失效链接和不清楚的操作步骤；
- 补充可复现的安装、兼容性和连接问题；
- 提交不包含敏感数据的功能建议；
- 改进 Issue 模板、版本记录和安全说明。

本仓库暂不接受客户端源代码、第三方二进制文件或未经授权的品牌素材。

## 提交 Issue 前

1. 确认使用的是 README 中列出的最新测试版本。
2. 搜索现有 Issue，避免重复报告。
3. 尽量在干净环境中重新复现问题。
4. 对日志和截图进行脱敏，删除账号、令牌、设备标识、IP 地址及其他隐私信息。
5. 安全漏洞请遵循 [SECURITY.md](./SECURITY.md)，不要创建公开 Issue。

## Bug 报告应包含

- 客户端平台和版本；
- 操作系统版本及硬件架构；
- 清晰的复现步骤；
- 预期行为和实际行为；
- 是否可以稳定复现；
- 已脱敏的必要日志或截图。

账号、订单、付款或必须提交隐私数据的问题，请改用[官网联系页面](https://www.phones-cloud.cn/contact)。

## 提交文档修改

1. Fork 本仓库并从 `main` 创建分支。
2. 保持每个 Pull Request 聚焦于一个主题。
3. 使用简洁、可操作的中文；必要时可以同时补充英文。
4. 不要把安装包、凭据、用户数据或内部配置提交到仓库。
5. 在 Pull Request 中说明修改原因，以及你验证过的系统或客户端版本。

提交贡献即表示你有权提供相关内容，并同意其作为本仓库公开文档的一部分展示。客户端软件的许可边界不因文档贡献而改变。

## 交流原则

请围绕可验证的问题交流，尊重不同系统、网络环境和技术背景的使用者。维护者可能会关闭无法复现、缺少必要信息、包含敏感数据或超出本仓库范围的 Issue。
